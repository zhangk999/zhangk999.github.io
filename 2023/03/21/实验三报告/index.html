实验题目：利用云平台搭建个人博客

实验日期：2023.3.15 实验学时：4

实验目的：

实验目的：熟悉个人博客系统的搭建。

具体包括：

参考方案：注册 Github 账号，搭建 Hexo 环境并实现个人博客搭建，撰写实验报告。

实验步骤与内容：

注册Github账号只需使用邮箱注册，不在此赘述。

Hexo环境搭建：

1. 安装Nodejs和Git，之前已经安装完成，在此不重复安装。****

2. *初始化hexo，新建存储博客的文件夹*

*hexo init myblog*

3、进入文件夹，安装一下npm

4、启动服务站点

*hexo g*

*hexo server*

*访问http://localhost:4000/ 至此hero就搭建好了。可以在本地访问了*

GitHub新建仓库：

停止localhost运行后，将hexo博客站点上传到gitee上

*这里需要安装一个hexo的上传插件deploy-git*

*npm install hexo-deployer-git –save*

*这里运行后出现错误，根据提示信息，键入npm audit fix –force即可解决。*

修改hexo配置文件指定仓库路径

*打开_config.yml文件，并在deploy处做如下修改*

*最后执行hexo d执行推送，就可以看到仓库中有推送上来的文件了*

*同时，也可以通过链接访问博客了*

博客主题更改:

在GitHub上搜索hexo主题*

fluid-dev/hexo-theme-fluid: 一款 Material Design 风格的 Hexo 主题 / An elegant Material-Design theme for Hexo (github.com)

*仓库给出详细的搭建步骤和用法，按照步骤照做即可。*

*更改并添加yml文件，并通过npm命令下载相关文件后，在博客目录下执行命令：*

*npm update –save hexo-theme-fluid。就可以在**Hexo (zhangk999.github.io)**查看到更改后的主题。*

*更改主题后：*

*在本地localhost:4000访问正常*

*最后，写文章，hexo new* *”**博客标题**”**，此命令会在source/_posts下创建.md文件，这就是要编写的文章。*

*在编写完毕后，通过下面三条命令即可将文件更新到GitHub。*

*hexo clean* *//清理*

*hexo g* *//构建静态文件*

*hexo d* *//上传至仓库*

*在第一篇博客.md编辑少量测试内容后，保存md文件，并执行上述三条命令(hexo d可能会因网络原因失败，多试几次)，可以看到GitHub文件有了同步更新。*

*打开博客网站，也有了新加入的博客*

处理Typora图片丢失问题：

*然后出现了图片丢失的问题，继续修改设置使用**Typora+Github+PicGo**实现自动上传图片*

*在Github再新建一个仓库，用来存放Markdown的图片，最终结果：*

最终效果：在博客页面可以正常访问图片，Github的访问有时较慢，需要有一些魔法，否则可能长时间加载不出图片。

结论分析与体会：

  1. 实验中通过Hexo快速搭建了自己的博客页面，也对Github的使用更加熟悉，对创建仓库等操作更加熟练和清晰。对云资源的便利和高效性认识更深。
  2. 搭建博客过程中遇到了Markdown中图片访问的问题，通过Typora+Github+PicGo的方式得到了很好的解决。这也正是云资源带来的便利。
  3. 通过实验熟悉了个人博客系统的搭建过程。同时复习了Markdown的一些语法。
